Learning

What is a Model?

In machine learning, a model is a mathematical function. It takes an input (an image, a number, a sentence) and produces an output (a category, a prediction, a score).

The simplest model you can imagine is a straight line:

y = w · x + b
x is the input, y is the output, and w (weight) and b (bias) are the model's parameters.

What are Parameters?

Parameters are the adjustable knobs of a model. Different parameter values produce different outputs for the same input. A model with w=2, b=1 behaves completely differently from one with w=-1, b=5.

What is Learning?

Learning is the process of finding parameter values that make the model's outputs match the desired outputs as closely as possible. We measure mismatch with a loss function (like Mean Squared Error), and learning means minimizing this loss.

Supervised Learning

Supervised learning is like learning with a teacher. The model is given:

  • Inputs — e.g., images of clothing items
  • Labels — e.g., "T-shirt", "Sneaker", "Dress"

The model learns to map inputs to the correct labels by adjusting its parameters to minimize the difference between its predictions and the true labels. The labels act as supervision — they tell the model what the right answer is.

Examples of supervised learning: Image classification, spam detection, speech recognition, medical diagnosis, predicting house prices.
3

Unsupervised Learning

What if we have data but no labels? Unsupervised learning finds hidden patterns and structures in unlabeled data.

The model examines the data on its own and discovers:

  • Clusters — groups of similar items
  • Patterns — underlying structure in the data
  • Anomalies — unusual data points that don't fit

No one tells the model what to look for — it finds structure by itself.

Examples: Customer segmentation, anomaly detection, topic discovery in documents, dimensionality reduction.
4

Self-Supervised Learning

Self-supervised learning is a clever trick: the model creates its own labels from the data itself.

How? By hiding part of the input and training the model to predict the hidden part:

  • Text: Mask a word in a sentence → predict the missing word. This is how GPT and BERT learn!
  • Images: Mask a patch of an image → predict the missing pixels.
  • Audio: Predict the next segment of a sound clip.

Because labels come from the data itself, models can learn from massive amounts of unlabeled data — the entire internet, for instance.

Key insight: Self-supervised learning bridges supervised and unsupervised learning. The data is unlabeled, but the model creates a supervised task for itself automatically.

Reinforcement Learning

Reinforcement learning is learning by trial and error, much like teaching a dog with treats.

An agent interacts with an environment:

  1. The agent observes a state (where it is)
  2. It chooses an action (where to go)
  3. The environment returns a reward (good or bad?)
  4. The agent updates its policy (strategy) to get more reward next time

Over many episodes of trial and error, the agent learns to make better decisions.

Examples: Game-playing AI (chess, Go, Atari), robotics, autonomous driving, resource management.

Deep Learning

Deep learning uses neural networks with many layers to learn complex patterns. A neural network is a model made of layers of interconnected "neurons," each performing a simple computation. What makes it powerful is the depth — stacking many layers.

Why Depth Matters

Each layer transforms the data into a more useful representation:

  • Early layers detect simple features (edges, colors, basic sounds)
  • Middle layers combine features into patterns (textures, shapes, phonemes)
  • Deep layers recognize high-level concepts (faces, objects, words)

A shallow network (one layer) can only learn simple, linear relationships. A deep network can learn increasingly abstract and complex representations.

Why "deep"? The word "deep" simply refers to the number of layers. A network with 2 hidden layers is deeper than one with 1. Modern networks (like GPT or image classifiers) can have hundreds of layers and billions of parameters.

Model Complexity

Model complexity refers to how flexible a model is — how many different patterns it can represent. A more complex model can capture more intricate relationships, but this flexibility comes with trade-offs.

What Controls Complexity?

  • Number of parameters — more weights and biases means more knobs to tune
  • Number of layers — deeper networks can represent more abstract features
  • Polynomial degree — a degree-1 polynomial is a line, degree-5 can wiggle, degree-15 can fit nearly anything
  • Model type — a linear model is simpler than a decision tree, which is simpler than a deep neural network
Key insight: More complexity is not always better. A model that is too simple can't learn the pattern (underfitting). A model that is too complex memorizes noise (overfitting). The goal is to find the right level of complexity for your data.

Bias

In machine learning, bias refers to the error introduced by approximating a complex real-world phenomenon with a simplified model. It measures how far off the model's predictions are from the true values, even on the training data.

High Bias = Underfitting

A model with high bias is too simple to capture the underlying pattern. It makes strong assumptions about the data (e.g., "the relationship is linear") and systematically misses the true structure. No matter how much training data you give it, it will keep making the same types of errors.

Think of it this way: Bias is the model's imperfection — its inability to represent the true relationship. A straight line trying to fit a curve will always have high bias, because it can't bend.

Variance

Variance measures how much the model's predictions change when you train it on different training sets. A high-variance model is very sensitive to which data it sees — small changes in the training set lead to wildly different fitted models.

High Variance = Overfitting

A complex model has the flexibility to fit not just the underlying pattern, but also the noise in each specific training set. When trained on a different sample, it fits that noise instead — producing a very different model each time.

Think of it this way: Train the same model on 5 different random samples from the same population. If the 5 fitted curves look similar → low variance. If they look wildly different → high variance.

Training & Validation Accuracy

As a model trains over many epochs (passes through the training data), we track two quantities:

  • Training accuracy — how well the model fits the data it's learning from
  • Validation accuracy — how well the model performs on held-out data it hasn't seen

The gap between these two curves tells an important story:

  • Both low → underfitting (model too simple)
  • Both high & close → good fit
  • Train high, val low/declining → overfitting (model memorizing noise)

Underfitting & Overfitting

This is the central tension in machine learning:

  • Underfitting — the model is too simple. It can't capture the true pattern. Both training and validation errors are high. (High bias, low variance)
  • Overfitting — the model is too complex. It memorizes the training data, including its noise. Training error is low but validation error is high. (Low bias, high variance)

The sweet spot is the model complexity where validation error is minimized — complex enough to learn the pattern, but not so complex that it memorizes noise.

The classic U-curve: As model complexity increases, training error always decreases. But validation error first decreases (less underfitting), hits a minimum (sweet spot), then increases (overfitting).

Cross Validation

A single train/validation split can be misleading — you might get lucky (or unlucky) with which data ends up in each set. K-Fold Cross Validation solves this by:

  1. Splitting the data into k equal parts (folds)
  2. Training k separate models, each time using a different fold as the validation set and the remaining k-1 folds for training
  3. Averaging the k validation scores to get a more reliable estimate
Why it matters: Cross validation gives you a robust estimate of how well your model generalizes, using all of your data for both training and validation. It reduces the variance of your performance estimate.

Bayes Error

The Bayes error rate is the lowest possible error rate achievable by any classifier, no matter how powerful. It represents the irreducible error — the noise inherent in the data itself.

Why Does Irreducible Error Exist?

Real-world data is noisy. Two identical-looking inputs might have different true labels (e.g., two very similar X-ray images, one with a tumor and one without). No model, no matter how complex, can distinguish these cases perfectly. This ambiguity is the Bayes error.

Error Decomposition

The total expected error of any model can be decomposed into three parts:

Total Error = Bias² + Variance + Bayes Error (irreducible noise)

Bias² — error from wrong assumptions (model too simple)
Variance — error from sensitivity to training data (model too complex)
Bayes Error — error that no model can eliminate (noise in the data)
