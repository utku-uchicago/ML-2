ML II — Group Activity: Autonomous AI Research
30:00 START
In-Class Activity

Can AI do research
by itself?

Explore Karpathy's autoresearch — an agent that autonomously trains, evaluates, and iterates on LLMs overnight.

30 minutes
Groups of 3–4
Google Colab

Overview

The big idea: autoresearch gives an AI agent a real LLM training setup and lets it modify code, run 5-minute experiments, check if results improved, and repeat — all night, without human input. Your group will read the code, run a mini-version, and discuss what this means for the future of ML research.

Activity Phases

PHASE 1
Setup & Orientation
0–7 min

Assign roles and get everyone oriented on what autoresearch actually does.

Driver
Navigator

Presenter

  • Navigator — reads the autoresearch README and directs what to run

No GPU needed for this activity — we'll only read and lightly modify the code, not run full training. Colab's free CPU is fine.
Open Google Colab
PHASE 2
Code Exploration
7–18 min

Clone the repo in Colab and explore the three key files. The goal isn't to run training — it's to understand how the agent loop works.

Python · Colab 
# Cell 1 — Clone the repo
!git clone https://github.com/karpathy/autoresearch
import os; os.chdir('autoresearch')

# Cell 2 — Inspect the three core files
!wc -l prepare.py train.py program.md   # how big are they?
!cat program.md                          # what does the agent read?

# Cell 3 — Find the agent's edit targets in train.py
!grep -n "DEPTH\|TOTAL_BATCH_SIZE\|learning_rate" train.py

# Cell 4 — Count how many hyperparams are exposed
!grep -c "^[A-Z_]* =" train.py
  • What does program.md tell the agent? Is it surprisingly short?
  • Which hyperparameters in train.py is the agent free to modify?
  • What is the single metric being optimized? Why val_bpb?
  • Why does every run use a fixed 5-minute time budget?
Python · Bonus — edit program.md 
# Cell 5 — Add one instruction to the agent
with open('program.md', 'a') as f:
    f.write("\n\n## Your Team's Constraint\n")
    f.write("Always try reducing DEPTH before changing anything else.\n")

# Then re-read to confirm
!tail -10 program.md
PHASE 3
Discussion
18–27 min

Step away from the screen. Each person picks one question to lead for 2 minutes.

Q 01
The agent has no memory across sessions. Every run starts fresh from train.py. Is that a feature or a bug?
Q 02
Results aren't comparable across GPUs. Does this undermine the scientific value of what the agent discovers?
Q 03
program.md is described as "programming the program." How is this different from a traditional system prompt?
Q 04
If you ran this overnight on an H100, would you trust the model changes it made? What would you check first?
Q 05
What kind of ML research can't be automated this way? Where does human intuition still matter?
Q 06
Karpathy calls this "the story of how it all began." Is he being ironic — or is this genuinely a preview of future research labs?
PHASE 4
Group Debrief
27–30 min

Each Presenter shares 2–3 takeaways with the class. Use the checklist below to make sure your group covered the key ideas.

  • Identified what program.md does and how to modify it
  • Listed at least 3 hyperparameters the agent can change in train.py
  • Explained why a fixed 5-min budget makes experiments comparable
  • Discussed one risk of fully autonomous code modification
  • Formed a group opinion: is this research automation or research replacement?
